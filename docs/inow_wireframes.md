# inow ワイヤーフレーム

- 対象: MVP
- 参照元: `docs/inow_screen_structure.md`, `docs/inow_specification.md`
- 目的: 主要画面の情報配置と遷移イメージをテキストベースで整理する

## 1. 方針

- モバイルアプリ想定の縦長レイアウトを前提とする
- `Map` 画面を中心に、地図を見る体験を最優先とする
- ワイヤーフレームは UI デザインではなく、要素配置と情報優先度の確認を目的とする

## 6. 補足

- デフォルトマップ `お気に入り` `行きたい` は `Profile` の保存タブおよび `Saved Maps` に表示する
- `Search` の主軸は `Map` 結果とし、`Spot` と `User` は切替タブで補助的に提供する
- `Add / Edit Spot` は Google Place 起点と手動作成起点の両方に対応する
- デザイン段階では FAB、検索バー、カード UI の優先度を高く扱う
