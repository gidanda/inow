# inow 仕様書

- 対象: MVP
- 参照元: `docs/inow_requirements.md`, `docs/inow_screen_structure.md`
- 目的: MVP 開発に必要な画面仕様、機能仕様、データ仕様、権限仕様を定義する

## 1. 仕様方針

### 1-1. サービスの基本単位
- 画面名としての `Map` はアプリのホーム画面を指す
- コンテンツ単位としての `マップ` は、複数のスポットを束ねたユーザー作成物を指す
- `スポット` はマップを構成する地点情報を指す

### 1-2. MVP の基本原則
- 主役はスポット単体ではなくマップ全体とする
- 公開範囲はマップ単位で管理し、スポット単位では管理しない
- 保存には `マップ保存` と `スポットを自分のマップへ追加` の 2 種類がある
- 初期体験は地図閲覧を中心に設計し、投稿導線は `Map` と `Profile` の文脈に置く

## 2. 用語定義

| 用語 | 定義 |
| --- | --- |
| マップ | ユーザーが作成する、複数スポットの集合 |
| スポット | マップに属する地点情報 |
| 保存マップ | 他ユーザーのマップを後で参照するために保存したマップ |
| デフォルトマップ | 自分用に自動作成される `お気に入り` と `行きたい` の 2 マップ |
| スポット保存 | 他人のスポットを自分のマップへ追加する行為 |

## 4. 認証・初回登録仕様

### 4-1. Login

#### 入力項目
| 項目 | 必須 | 備考 |
| --- | --- | --- |
| メールアドレス | 必須 | メール形式 |
| パスワード | 必須 | 8 文字以上を想定 |

#### 挙動
- 認証成功時は `Map` に遷移する
- 未認証ユーザーの場合は `Verification` に誘導する
- 認証失敗時はエラーメッセージを表示する

### 4-2. Sign Up

#### 入力項目
| 項目 | 必須 | 備考 |
| --- | --- | --- |
| メールアドレス | 必須 | 重複不可 |
| パスワード | 必須 | 8 文字以上 |

#### 挙動
- 登録成功時は認証コード送信後に `Verification` へ遷移する
- 同一メールアドレスが存在する場合は登録不可

### 4-3. Verification

#### 入力項目
| 項目 | 必須 | 備考 |
| --- | --- | --- |
| 認証コード | 必須 | 数字または英数字を想定 |

#### 挙動
- 認証成功時は `Onboarding` に遷移する
- 認証コード再送を可能とする
- 失敗時は再入力を促す

### 4-4. Onboarding

#### 画面構成
以下の項目を 1 画面ずつ入力させる。

| 順序 | 項目 | 必須 | 備考 |
| --- | --- | --- | --- |
| 1 | 姓 | 必須 | 50 文字以内 |
| 2 | 名 | 必須 | 50 文字以内 |
| 3 | 生年月日 | 必須 | 日付形式 |
| 4 | ユーザー ID | 必須 | 英数字と `_`、重複不可 |
| 5 | 表示名 | 必須 | 30 文字以内 |
| 6 | プロフィール画像 | 任意 | 選択または撮影 |

#### 挙動
- 全項目完了後にユーザー登録完了とし、`Map` を初期表示する
- `お気に入り` と `行きたい` のデフォルトマップを自動生成する

## 5. 画面仕様

### 5-1. Map

#### 目的
現在地周辺や閲覧中の範囲に存在するスポットを地図上で確認する。

#### 表示要素
- 検索バー
- フィルターボタン
- 地図
- スポットピン
- ピン選択時のプレビューカード
- スポット追加 FAB
- マップ作成導線
- 下部ナビゲーション

#### 主な挙動
- 初回表示時は現在地取得を試行する
- 現在地取得を許可しない場合はデフォルトエリアを表示する
- ピン選択時はスポットの簡易情報をプレビュー表示する
- プレビュータップで `Spot Detail` に遷移する
- 地図長押しまたは FAB で `Add / Edit Spot` に遷移する
- マップ作成導線から `Add / Edit Map` に遷移する

#### 空状態
- 該当エリアにスポットがない場合は「このエリアにはまだスポットがありません」を表示する

### 5-2. Search

#### 表示要素
- 検索入力欄
- 検索履歴
- 人気タグ
- カテゴリ一覧
- 結果一覧
- 地図表示 / 一覧表示切替
- フィルターボタン

#### 検索対象
- スポット名
- マップタイトル
- マップタグ
- エリア
- ユーザー表示名
- ユーザー ID

#### 主な挙動
- キーワード未入力時は履歴と人気タグを表示する
- ユーザー結果選択時は `User Profile` に遷移する
- スポット結果選択時は `Spot Detail` に遷移する
- 一覧表示と地図表示は同一検索条件を共有する

#### 空状態
- 該当結果がない場合は「条件に一致する結果がありません」を表示する

### 5-3. Friends

#### 表示要素
- フォロー中ユーザー一覧
- アイコン
- ユーザー名
- 簡易プロフィール
- 得意エリア
- よく使うタグ
- 最近公開したマップの要約

#### 主な挙動
- フォロー中ユーザーのみ表示する
- ユーザー選択で `User Profile` に遷移する
- 未フォロー時は空状態を表示する

#### 空状態
- 「まだフォロー中のユーザーがいません」を表示する
- `Search` への導線を表示する

### 5-4. Profile

#### 表示要素
- アイコン
- 表示名
- 自己紹介
- 作成マップ数
- 保存マップ数
- 得意エリア
- よく使うタグ
- `自分のマップ` タブ
- `保存したマップ` タブ
- `新しいマップを作る` ボタン
- `設定` ボタン

#### 主な挙動
- `自分のマップ` タブでは自分の作成マップを表示する
- `保存したマップ` タブでは保存マップとデフォルトマップを表示する
- マップ選択で `User Map` に遷移する
- 作成ボタンで `Add / Edit Map` に遷移する

### 5-5. User Profile

#### 表示要素
- アイコン
- 表示名
- 自己紹介
- フォロー / フォロー解除ボタン
- 公開マップ一覧
- 得意エリア
- よく使うタグ

#### 主な挙動
- 非公開マップは表示しない
- 公開マップ選択で `User Map` に遷移する
- フォロー状態を即時反映する

### 5-6. User Map

#### 表示要素
- マップタイトル
- 説明
- 作成者
- カバー画像
- タグ
- 地図
- スポット一覧
- マップ保存ボタン
- いいねボタン
- コメント導線
- 自分のマップの場合は編集ボタン

#### 主な挙動
- 選択された 1 マップのみを表示する
- スポット一覧と地図ピンは同じデータを参照する
- スポット選択で `Spot Detail` に遷移する
- 他人の公開マップは保存できる
- 自分のマップは保存対象外とする

#### 空状態
- スポットが 0 件の場合は「まだスポットがありません」を表示する

### 5-7. Spot Detail

#### 表示要素
- スポット名
- 写真
- 説明コメント
- 位置情報
- タグ
- 投稿元マップ情報
- 投稿者情報
- `自分のマップに追加` ボタン
- 自分の投稿の場合は編集ボタン

#### 主な挙動
- `自分のマップに追加` 選択時は追加先マップ選択モーダルまたは画面を表示する
- 追加先候補には自分の作成マップと `お気に入り` `行きたい` を表示する
- 追加確定時は新しいスポットとして自分のマップに複製登録する
- 投稿者選択で `User Profile` に遷移する
- 投稿元マップ選択で `User Map` に遷移する

### 5-8. Saved Maps

#### 表示要素
- 保存した他ユーザーのマップ一覧
- `お気に入り`
- `行きたい`

#### 主な挙動
- 他ユーザーの保存マップとデフォルトマップを同一画面で表示する
- マップ選択で `User Map` に遷移する
- 他ユーザーの保存マップのみ保存解除できる

### 5-9. Add / Edit Map

#### 入力項目
| 項目 | 必須 | 備考 |
| --- | --- | --- |
| タイトル | 必須 | 1 文字以上 100 文字以内 |
| 説明 | 任意 | 500 文字以内 |
| カバー画像 | 任意 | 1 枚 |
| タグ | 任意 | 最大 10 件 |
| 公開設定 | 必須 | `公開` / `非公開` |

#### 主な挙動
- 新規作成時は空フォームを表示する
- 編集時は既存値を初期表示する
- 保存成功後は対象の `User Map` に遷移する
- 削除は編集時のみ可能とする

### 5-10. Add / Edit Spot

#### 入力項目
| 項目 | 必須 | 備考 |
| --- | --- | --- |
| 追加先マップ | 必須 | 自分のマップのみ選択可能 |
| スポット名 | 必須 | 1 文字以上 100 文字以内 |
| 位置情報 | 必須 | 地図選択または座標確定 |
| コメント | 任意 | 500 文字以内 |
| 写真 | 任意 | 複数枚対応は将来拡張、MVP は最大 1 枚 |
| タグ | 任意 | 最大 10 件 |

#### 主な挙動
- `Map` から遷移した場合、長押し位置を初期座標として設定する
- 編集時は既存値を初期表示する
- 保存成功後は追加先 `User Map` に戻る
- 削除は編集時のみ可能とする

### 5-11. Filter

#### 条件項目
| 項目 | 適用先 |
| --- | --- |
| 現在地からの距離 | Map, Search |
| エリア | Search |
| タグ | Map, Search |
| カテゴリ | Search |

#### 主な挙動
- 条件は適用元画面へ戻る際に即時反映する
- リセット操作で全条件を初期化する

### 5-12. Settings

#### 項目
- プロフィール編集
- 通知設定
- ブロック管理
- ログアウト

## 7. 保存・追加仕様

### 7-1. マップ保存
- 対象は他ユーザーの公開マップのみ
- 保存先はユーザーの `Saved Maps`
- 自分のマップは保存できない

### 7-2. スポット保存
- `Spot Detail` から実行する
- 処理内容は「既存スポットを参照する」のではなく「自分のマップへ追加する」
- 追加先は `お気に入り` `行きたい` を含む自分のマップから選択する

### 7-3. デフォルトマップ
- ユーザー登録完了時に `お気に入り` `行きたい` を自動生成する
- 削除不可とする
- 名前変更は MVP では不可とする

## 8. 権限仕様

### 8-1. 未ログインユーザー
- 公開マップ閲覧: 可
- 検索: 可
- 他ユーザープロフィール閲覧: 可
- マップ作成、スポット追加、保存、いいね、コメント、フォロー: 不可

### 8-2. ログイン済みユーザー
- 自分のマップ / スポットの作成、編集、削除: 可
- 他ユーザーの公開マップ閲覧: 可
- 他ユーザーの公開マップ保存: 可
- 他ユーザーのスポットを自分のマップへ追加: 可

### 8-3. 公開範囲
- `public`: 全ユーザーが閲覧可能
- `private`: 作成者のみ閲覧可能

## 9. バリデーション仕様

### 9-1. 共通
- 必須項目未入力時は保存不可
- 文字数超過時はフィールド単位でエラー表示する
- ネットワークエラー時は送信失敗メッセージを表示する

### 9-2. ユーザー登録
- メールアドレス形式不正はエラー
- 重複メールアドレスはエラー
- 重複ユーザー ID はエラー

### 9-3. マップ作成
- タイトル未入力はエラー
- 公開設定未選択は保存不可

### 9-4. スポット作成
- 追加先マップ未選択はエラー
- スポット名未入力はエラー
- 位置情報未確定は保存不可

## 10. エラー・空状態仕様

### 10-1. エラー
- 認証失敗: 入力内容の見直しを促す
- 保存失敗: 再試行導線を表示する
- 権限なし: ログイン導線または閲覧不可メッセージを表示する
- 削除失敗: 対象が見つからないか権限不足として扱う

### 10-2. 空状態
- Map: スポットなしメッセージ
- Search: 検索結果なしメッセージ
- Friends: フォローなしメッセージ
- Profile: 作成マップなし / 保存マップなしメッセージ
- User Profile: 公開マップなしメッセージ

## 11. 画面遷移の要点

### 11-1. 作成導線
- `Map` → `Add / Edit Spot`
- `Map` → `Add / Edit Map`
- `Profile` → `Add / Edit Map`
- `Spot Detail` → `Add / Edit Spot`

### 11-2. 閲覧導線
- `Map` → `Spot Detail`
- `Search` → `Spot Detail`
- `Search` → `User Profile`
- `Friends` → `User Profile`
- `User Profile` → `User Map`
- `Saved Maps` → `User Map`
