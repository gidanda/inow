# inow 画面構成書

- 対象: MVP 想定
- 目的: 仕様書作成時の基礎資料として、画面一覧・役割・遷移・主要要素を整理する

## 1. 画面構成の全体方針

`inow` は、地図を中心に「場所を知る」「人の知っている場所を見る」「自分の地図を持つ」ことを主目的とするサービスである。
そのため、画面構成は投稿中心ではなく、地図中心の導線を採用する。

本書における用語は以下のように整理する。

- `Map` 画面: アプリのホームとして地図 UI を表示する画面名
- マップ: ユーザーが作成する、複数スポットの集合としてのコンテンツ単位
- スポット: マップ上に追加される地点情報

MVP では、下部ナビゲーションを以下の 4 画面で構成する。

- `Map`
- `Search`
- `Friends`
- `Profile`

また、これらに加えてサブ画面として以下を持つ。

- `Spot Detail`
- `Filter`
- `User Profile`
- `User Map`
- `Saved Maps`
- `Add / Edit Spot`
- `Add / Edit Map`
- `Settings`
- `Login`
- `Sign Up`
- `Verification`
- `Onboarding`

## 2. 画面一覧

### 2-1. メイン画面一覧

| 画面名 | 役割 | 備考 |
| --- | --- | --- |
| Map | ホーム画面。地図上でスポットを閲覧・操作する中心画面 | 最重要画面 |
| Search | 条件・キーワード・タグ・ユーザーから検索する画面 | 探索用 |
| Friends | フォロー中ユーザー一覧、および各ユーザーの地図への導線 | 人起点の導線 |
| Profile | 自分の情報、自分の作成マップ、保存したマップ、設定への導線 | 自分起点の導線 |

### 2-2. サブ画面一覧

| 画面名 | 役割 |
| --- | --- |
| Spot Detail | スポットの詳細情報を表示する |
| Filter | Map / Search に対する絞り込み条件を設定する |
| User Profile | 他ユーザーのプロフィールと公開マップを表示する |
| User Map | 特定のマップを地図上で表示する |
| Saved Maps | 保存したマップ、およびデフォルトマップを表示する |
| Add / Edit Spot | スポットの新規追加・編集を行う |
| Add / Edit Map | マップの新規作成・編集を行う |
| Settings | アカウント・通知などの設定を行う |
| Login | ログインを行う |
| Sign Up | 新規登録を開始する |
| Verification | メール認証コードの入力を行う |
| Onboarding | 初回登録時にユーザー情報を 1 画面ずつ入力する |

## 3. ナビゲーション構成

### 3-1. 下部ナビゲーション

下部タブバーに以下 4 項目を配置する。

- `Map`
- `Search`
- `Friends`
- `Profile`

### 3-2. 基本導線

- アプリ起動時は `Map` を初期表示とする
- 未ログイン時は必要に応じて `Login` / `Sign Up` / `Verification` / `Onboarding` に遷移する
- `Map` からスポットを選択すると `Spot Detail` に遷移する
- `Search` の検索結果からスポットを選択すると `Spot Detail` に遷移する
- `Search` や `Friends` からユーザーを選択すると `User Profile` に遷移する
- `User Profile` から対象ユーザーの公開マップを選択すると `User Map` に遷移する
- `Profile` から自分の作成マップ、保存マップ、設定に遷移できる
- スポット追加は独立タブではなく、`Map` / `Profile` / `User Map` 内の操作から行う
- マップ作成は `Map` または `Profile` から `Add / Edit Map` に遷移して行う

## 4. 各画面の詳細

### 4-1. Map

#### 目的
地図上でスポットを閲覧し、`inow` の中核体験を提供するホーム画面。

#### 主な機能
- 現在地周辺のスポット表示
- 地図移動・拡大縮小
- スポットピンの表示
- ピン選択によるスポット情報プレビュー
- `Spot Detail` への遷移
- フィルター適用
- スポット追加
- マップ作成導線
- 自分のマップ編集導線

#### 主な UI 要素
- 上部検索バー
- フィルターボタン
- 地図表示エリア
- スポットピン
- 下部スポットプレビューカード
- 右下追加ボタン（FAB 想定）
- マップ作成ボタンまたはメニュー導線
- 下部ナビゲーション

#### 想定アクション
- 地図を移動する
- ピンをタップする
- 地図を長押ししてスポット追加する
- フィルターを開く
- 検索画面へ移動する
- 新しいマップを作成する

#### 遷移先
- `Spot Detail`
- `Filter`
- `Add / Edit Spot`
- `Add / Edit Map`
- `Search`

#### 備考
`Map` はアプリ全体のホームであり、最も使用頻度が高い画面とする。

### 4-2. Search

#### 目的
スポット・エリア・タグ・ユーザーなどを条件から探す画面。

#### 主な機能
- キーワード検索
- 地名検索
- スポット名検索
- タグ検索
- カテゴリ検索
- ユーザー検索
- 検索結果の一覧表示
- 検索結果の地図表示
- フィルター設定

#### 主な UI 要素
- 検索入力欄
- 最近の検索履歴
- 人気タグ
- カテゴリ一覧
- ユーザー検索結果
- スポット検索結果
- 地図表示 / 一覧表示切替
- フィルターボタン
- 下部ナビゲーション

#### 想定アクション
- 検索語を入力する
- タグを選択する
- 結果一覧からスポットを選ぶ
- 結果一覧からユーザーを選ぶ
- 地図表示に切り替える

#### 遷移先
- `Spot Detail`
- `User Profile`
- `Filter`

#### 備考
`Map` は「見る」導線、`Search` は「探しにいく」導線として役割を分ける。

### 4-3. Friends

#### 目的
フォロー中ユーザーを一覧で確認し、その人の持つ公開マップを見るための入口とする。

#### 主な機能
- フォロー中ユーザー一覧表示
- ユーザー検索導線
- 各ユーザーの `User Profile` への遷移
- ユーザーごとの得意エリア・タグ確認
- 最近公開したマップの確認

#### 主な UI 要素
- フォロー中ユーザー一覧
- ユーザーごとのアイコン
- ユーザー名
- 簡易プロフィール
- 得意エリア表示
- よく使うタグ表示
- 小さなマッププレビュー
- 下部ナビゲーション

#### 想定アクション
- フレンド一覧をスクロールする
- 特定ユーザーを選択する
- ユーザーのプロフィールを見る
- ユーザーの公開マップを見る

#### 遷移先
- `User Profile`

#### 備考
`Friends` は単なる SNS の友達一覧ではなく、人を起点に地図を見る導線として設計する。

### 4-4. Profile

#### 目的
自分の情報、自分が作成したマップ、保存したマップなどを管理する画面。

#### 主な機能
- 自分のプロフィール表示
- 自分の作成マップ一覧表示
- 保存マップ一覧表示
- デフォルトマップである「お気に入り」「行きたい」への導線
- マップ作成導線
- `Settings` への導線

#### 主な UI 要素
- アイコン
- ユーザー名
- 自己紹介
- 自分の作成マップ数
- 保存マップ数
- 自分の得意エリア
- よく使うタグ
- 「自分のマップ」タブ
- 「保存したマップ」タブ
- 「新しいマップを作る」ボタン
- 「設定」ボタン
- 下部ナビゲーション

#### 想定アクション
- 自分のマップを見る
- 保存済みマップを見る
- デフォルトマップを開く
- 新しいマップを作る
- 既存マップを編集する
- アカウント設定を開く

#### 遷移先
- `User Map`（自分自身のマップ）
- `Saved Maps`
- `Add / Edit Map`
- `Settings`

#### 備考
`Profile` は単なる設定画面ではなく、自分の地図と場所知識を管理する画面とする。

## 5. サブ画面詳細

### 5-1. Spot Detail

#### 目的
選択したスポットの詳細情報を表示する。

#### 主な機能
- スポット名表示
- 位置情報表示
- 写真表示
- 投稿者表示
- タグ表示
- コメント表示
- 自分のマップへの追加
- 編集（自分の投稿のみ）
- 投稿元マップへの遷移
- 投稿者プロフィールへの遷移

#### 遷移元
- `Map`
- `Search`
- `User Map`

#### 遷移先
- `Add / Edit Spot`
- `User Map`
- `User Profile`
- `Profile`

### 5-2. Filter

#### 目的
スポット表示や検索結果の絞り込み条件を設定する。

#### 想定項目
- 現在地からの距離
- エリア
- タグ
- カテゴリ

#### 遷移元
- `Map`
- `Search`

#### 遷移先
- `Map`
- `Search`

### 5-3. User Profile

#### 目的
他ユーザーのプロフィールと、公開されているマップ一覧を表示する。

#### 主な機能
- ユーザー情報表示
- フォロー / フォロー解除
- 公開マップ一覧表示
- 得意エリア、タグ表示
- 各マップの `User Map` への遷移

#### 遷移元
- `Search`
- `Friends`
- `Spot Detail`

#### 遷移先
- `User Map`

### 5-4. User Map

#### 目的
特定の 1 マップを地図上で表示し、そのマップに含まれるスポットを閲覧する。

#### 主な機能
- マップタイトル、説明、作成者表示
- マップ内スポット一覧表示
- 地図上でのスポット表示
- `Spot Detail` への遷移
- マップ保存
- 自分のマップの場合は編集導線表示

#### 遷移元
- `User Profile`
- `Profile`
- `Saved Maps`
- `Spot Detail`

#### 遷移先
- `Spot Detail`
- `Add / Edit Map`

### 5-5. Saved Maps

#### 目的
保存した他ユーザーのマップ、および自分のデフォルトマップを一覧で確認する。

#### 主な機能
- 保存マップ一覧表示
- デフォルトマップ「お気に入り」「行きたい」の表示
- 各マップの詳細表示
- 保存解除

#### 補足
「お気に入り」「行きたい」は自分用のデフォルトマップとして用意し、スポット保存時はこれらまたは任意の自分のマップに追加する。

#### 遷移元
- `Profile`

#### 遷移先
- `User Map`

### 5-6. Add / Edit Spot

- 保存

#### 遷移元
- `Map`
- `User Map`
- `Spot Detail`

#### 主な機能
- メールアドレス入力

#### 目的

#### 主な機能

#### 目的
メール認証コードを入力し、登録を有効化する。

#### 主な機能

#### 目的
初回登録時に必要なユーザー情報を 1 画面ずつ入力する。

#### 入力項目

- 生年月日
- ユーザー ID
- 表示名
- プロフィール画像の選択または撮影

#### 備考
- 各入力項目ごとに 1 画面を用意する
- 入力順は仕様書で最適化する

## 6. 主要ユーザーフロー

### 6-1. スポット閲覧フロー
1. `Map` を開く
2. 地図上のピンをタップする
3. `Spot Detail` を確認する
4. 自分のマップへ追加する、または投稿者情報へ進む

### 6-2. 検索フロー
1. `Search` を開く
2. キーワードやタグを入力する
3. 結果一覧を確認する
4. `Spot Detail` または `User Profile` へ遷移する

### 6-3. フレンド経由フロー
1. `Friends` を開く
2. フォロー中ユーザーを選択する
3. `User Profile` を確認する
4. 公開マップを選択して `User Map` を開く
5. 気になるスポットを `Spot Detail` で確認する

### 6-4. スポット追加フロー
1. `Map` 上で追加ボタンを押す、または地図を長押しする
2. `Add / Edit Spot` を開く
3. 追加先マップを選択する
4. 必要情報を入力する
5. 保存する
6. 自分のマップに反映する

### 6-5. マップ作成フロー
1. `Map` または `Profile` からマップ作成を選ぶ
2. `Add / Edit Map` を開く
3. タイトル、説明、公開範囲などを入力する
4. 保存する
5. `User Map` で内容を確認する

### 6-6. 自分の管理フロー
1. `Profile` を開く
2. 自分のマップまたは保存したマップを切り替える
3. 必要に応じて編集・確認する

### 6-7. 初回登録フロー
1. `Sign Up` で登録を開始する
2. `Verification` で認証を完了する
3. `Onboarding` でユーザー情報を 1 画面ずつ入力する
4. 完了後に `Map` を初期表示する

## 9. 補足方針

- `Add` は独立したメインメニューにしない
- 追加行為は `Map` / `Profile` の文脈の中で行う
- `inow` のコア体験は「地図を見ること」に置く
- `Friends` は「人から地図を見る」ための導線として扱う
- `Profile` は設定だけでなく、自分の場所知識を管理する画面とする
- 保存には「他人のマップを保存する」と「スポットを自分のマップへ追加する」の 2 種類がある
